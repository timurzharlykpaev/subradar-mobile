# Wiki Schema

Спецификация структуры и правил ведения LLM Wiki для проекта SubRadar Mobile.

## Структура каталогов

```
wiki/
  SCHEMA.md          # Этот файл — соглашения и правила
  index.md           # Индекс всех страниц, сгруппированных по категориям
  log.md             # Хронологический лог изменений
  pages/             # Страницы знаний (контент)
  sources/           # Внешние материалы, заметки по source-файлам (опционально)
  assets/            # Диаграммы, схемы в текстовом формате (опционально)
```

## Формат страницы

Каждый файл в `pages/` — Markdown с YAML frontmatter:

```yaml
---
title: "Заголовок страницы"
tags: [архитектура, стейт, zustand]
sources:
  - src/stores/settingsStore.ts
  - docs/STATE_RULES.md
updated: 2026-04-16
---
```

### Обязательные поля frontmatter

| Поле | Тип | Описание |
|------|-----|----------|
| `title` | string | Краткий заголовок |
| `tags` | string[] | Теги для поиска и группировки |
| `sources` | string[] | Файлы-источники из кодовой базы |
| `updated` | date | Дата последнего обновления (YYYY-MM-DD) |

## Соглашения по именам

- Файлы: **kebab-case** (`currency-system.md`, `ai-features.md`)
- Без пробелов, без заглавных букв в именах файлов
- Каждая страница — одна концепция/сущность/фича

## Кросс-ссылки

Используется Obsidian-синтаксис: `[[page-name]]` (без расширения `.md`).

Примеры:
- `[[state-management]]` — ссылка на страницу state-management.md
- `[[currency-system]]` — ссылка на страницу currency-system.md
- `[[subscriptions]]` — ссылка на страницу subscriptions.md

## Воркфлоу

### Ingest (добавление знаний)

1. Определи, к какой странице относится новая информация
2. Если страницы нет — создай в `pages/` с frontmatter
3. Обнови `sources` в frontmatter
4. Обнови `updated` дату
5. Добавь запись в `log.md`
6. Обнови `index.md` если добавлена новая страница

### Query (поиск знаний)

1. Начни с `index.md` — найди нужную категорию
2. Перейди на страницу
3. Следуй кросс-ссылкам `[[page-name]]` при необходимости

### Lint (проверка целостности)

Периодически проверяй:
- [ ] Все страницы из `pages/` перечислены в `index.md`
- [ ] Все `[[ссылки]]` ведут на существующие страницы
- [ ] Поля `sources` указывают на реальные файлы в репозитории
- [ ] Дата `updated` соответствует реальности
- [ ] Нет дублирования информации между страницами

## Правила обновления index.md

- Группировка по категориям (Архитектура, Сущности, Экраны, API, Фичи, Инфраструктура)
- Каждая страница — одна строка с кратким описанием
- Новая страница добавляется в соответствующую категорию

## Правила обновления log.md

Формат записи:

```markdown
## YYYY-MM-DD — Краткое описание
- Добавлено/обновлено: список страниц
- Причина: контекст изменений
```

Записи идут в обратном хронологическом порядке (новые сверху).
